## 章节导言

社区分享的 JSON 工作流是最好的学习素材，但要让它真正成为自己的生产力，需要完成三个动作：正确导入、按需求调节、遇错能诊断。本章将这三个环节拆成步骤，帮你从“复制粘贴”进阶到“吸收整合”。

## 阶段一：发现并导入工作流

<StepCard index={1} title="下载并导入" summary="拖放到画布即可加载" id="reuse-1">
  <ol>
    <li>在 RunningHub、Civitai 等社区下载心仪的工作流 JSON。</li>
    <li>在 ComfyUI 画布空白处拖入文件，或使用顶部菜单的 <strong>Load</strong> 功能。</li>
    <li>先整体浏览作者的注释，标记出模型路径、LoRA 设置等关键节点。</li>
  </ol>
</StepCard>

![导入社区工作流示意](/assets/uploads/wechat_2025-10-05_134655_350.png)
![阅读作者注释示意](/assets/uploads/wechat_2025-10-05_134816_997.png)

## 阶段二：识别可调节点

<StepCard index={2} title="替换模型与 LoRA" summary="确保风格可控" id="reuse-2">
  <ol>
    <li>优先检查 <code>Checkpoint Loader</code> 或 <code>UNet Loader</code>，替换为自己本地可用的模型。</li>
    <li>核对所有 <code>LoRA Loader</code> 是否存在于你的 LoRA 文件夹，并根据需求调整权重。</li>
    <li>确认 <code>VAE Loader</code> 是否仍指向作者机器，如果缺失就换成默认 VAE。</li>
  </ol>
</StepCard>

<StepCard index={3} title="重写提示词" summary="先复刻再扩展" id="reuse-3">
  <ol>
    <li>复制原作者的正向/反向提示词，为后续对比保留基线。</li>
    <li>先做最小改动：替换主体、补充环境描述、删除无关修饰。</li>
    <li>保留常见的负向词（如 <code>low quality</code>、<code>blurry</code>），确保生成质量稳定。</li>
  </ol>
</StepCard>

<StepCard index={4} title="检查执行参数" summary="KSampler 决定稳定度" id="reuse-4">
  <ol>
    <li>确认 Steps、CFG、采样器与调度器是否合理，避免过高或过低导致发散。</li>
    <li>决定是否固定 Seed 值，方便和原作输出对比。</li>
    <li>检查输出节点是否仍连接到 <code>Save Image</code>，避免图像只存在内存。</li>
  </ol>
</StepCard>

## 阶段三：诊断并解决报错

<StepCard index={5} title="配合 LLM 诊断" summary="提问要具体" id="reuse-5">
  <ol>
    <li>遇到红色报错时，先在 ComfyUI 控制台复制完整的错误日志。</li>
    <li>将日志与工作流 JSON 一同提交给 Gemini、ChatGPT 或 Claude，说明当前目标。</li>
    <li>根据回答逐条验证：是否缺少插件、模型路径是否错误、参数是否超界。</li>
  </ol>
</StepCard>

![诊断示意一](/assets/uploads/wechat_2025-10-05_140726_100.png)
![诊断示意二](/assets/uploads/wechat_2025-10-05_140837_984.png)

<TipCard title="常见排查速查表" tone="warning">
  <ul>
    <li><strong>Model not found</strong>：路径指向作者本地，重新指定即可。</li>
    <li><strong>Missing input</strong>：多数是采样器缺少 <code>LATENT</code> 或 LoRA 未连接。</li>
    <li><strong>Python package error</strong>：检查是否缺少自定义节点插件，或先禁用相关节点。</li>
  </ul>
</TipCard>

## 附：诊断提示词模板

```markdown
# Role: ComfyUI 工作流分析师

## Profile
- author: LangGPT
- version: 1.0
- language: 中文
- description: 你是一位熟练的 ComfyUI 工作流分析师，擅长分析上传的 JSON，快速识别核心功能、关键插件及参数对效果的影响。

## Skills
1. 解析 ComfyUI JSON 文件结构，提取核心工作流信息。
2. 分析工作流的主要功能模块和目标。
3. 识别实现这些功能所需的核心插件及其参数。
4. 评估参数调整对最终效果的潜在影响。

## Goals
- 明确工作流的核心功能和目标（如图像合成、风格化或去噪）。
- 列出实现这些功能的核心插件及其作用。
- 找出在工作流中起关键作用的参数，并说明调整后的影响。

## Output Format
- 核心功能: ...
- 核心插件: ...
- 关键参数: ...
- 备注: ...

## Rules
1. 输出条理清晰，突出“核心”要素。
2. 使用简洁的语句和条列式结构。
3. 根据上传的 JSON 给出实用建议。

## Workflow
1. 解析 JSON 节点与连线。
2. 概括整体流程与目标。
3. 提炼关键插件和参数。
4. 结合目标给出优化建议。
5. 输出结构化总结供用户参考。
```

按照以上流程，你就能把社区工作流改造成个人常用模板，并且遇到报错时也有能力自救。
