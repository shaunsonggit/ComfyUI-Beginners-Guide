## 章节导言

目标很明确：我们要用最少的节点搭建一条稳定的 Qwen-Image 文生图工作流。完成本章后，你能手动替换提示词、调整分辨率，并将结果保存到本地，而不再依赖别人分享的现成图。

<TipCard title="动手前的 3 项检查" tone="info">
  <ul>
    <li>模型文件夹不要包含中文路径，文件名需与 ComfyUI 设置一致。</li>
    <li>提前写好正向/反向提示词，避免边构建边思考文案。</li>
    <li>准备好输出目录，并确认有足够磁盘空间存放生成图片。</li>
  </ul>
</TipCard>

## 阶段一：搭好基础积木

<StepCard index={1} title="加载模型底座" summary="Checkpoint、LoRA、VAE、CLIP 四件套" id="text2img-1">
  <ol>
    <li>添加 <code>Checkpoint Loader</code>，选择你准备好的基础模型。</li>
    <li>若需要额外风格，在下游接入 <code>LoRA Loader</code> 并设置合适的权重。</li>
    <li>放置 <code>VAE Loader</code> 与 <code>CLIP Loader</code>，分别输出 <code>VAE</code> 与 <code>CLIP</code> 信号。</li>
  </ol>
</StepCard>

![模型加载节点示意](/assets/uploads/wechat_2025-10-05_122119_024.png)

<StepCard index={2} title="准备提示词" summary="正向 + 反向共同决定风格" id="text2img-2">
  <ol>
    <li>添加两个 <code>CLIP Text Encode</code> 节点，分别填入正向与负向提示词。</li>
    <li>确保它们的 <code>clip</code> 输入端来自同一个 <code>CLIP Loader</code>，以便共享词向量空间。</li>
    <li>常见负向词：<code>blurry</code>、<code>low quality</code>、<code>extra limbs</code>。</li>
  </ol>
</StepCard>

![提示词节点示意](/assets/uploads/wechat_2025-10-05_124132_373.png)

<StepCard index={3} title="建立潜空间画布" summary="Empty Latent Image 决定分辨率" id="text2img-3">
  <ol>
    <li>设置宽高为 1024 × 1536，保持 64 的倍数可避免报错。</li>
    <li>批次大小设为 1，后续想要批量生成再做扩展。</li>
    <li>随机种子可暂时留空，后续通过 KSampler 控制。</li>
  </ol>
</StepCard>

![潜空间节点示意](/assets/uploads/wechat_2025-10-05_124425_542.png)

## 阶段二：闭合生成环路

<StepCard index={4} title="配置 KSampler" summary="四条输入缺一不可" id="text2img-4">
  <ol>
    <li>将 <code>MODEL</code>（基础模型 + LoRA）接入 KSampler 的 <code>model</code> 端口。</li>
    <li>把正向、负向提示词分别接入 <code>positive</code> 与 <code>negative</code>。</li>
    <li>将 <code>Empty Latent Image</code> 输出的 <code>latent</code> 接到 KSampler。</li>
    <li>初始参数建议：Steps 20、CFG 7、Sampler = Euler、Scheduler = Simple。</li>
  </ol>
</StepCard>

![采样器设置示意](/assets/uploads/wechat_2025-10-05_130249_357.png)

<StepCard index={5} title="解码并保存" summary="VAE Decode + Save Image" id="text2img-5">
  <ol>
    <li>添加 <code>VAE Decode</code>，将 KSampler 输出的潜空间连到 <code>latent</code>，VAE Loader 连接到 <code>vae</code>。</li>
    <li>再添加 <code>Save Image</code>，把解码后的 <code>image</code> 输出接入其中。</li>
    <li>自定义文件名前缀，例如 <code>qwen_image_</code>，方便管理。</li>
  </ol>
</StepCard>

![保存节点示意](/assets/uploads/wechat_2025-10-05_130936_692.png)

<TipCard title="快速验证流程" tone="success">
  <ol>
    <li>点击运行，确认控制台没有红色报错。</li>
    <li>打开输出目录，检查生成结果是否符合提示词描述。</li>
    <li>若需要复现，请记录 KSampler 下方显示的 Seed 值。</li>
  </ol>
</TipCard>

![运行效果示意](/assets/uploads/wechat_2025-10-05_131524_185.png)

## 阶段三：扩展与复用

- **换提示词**：保留原有节点，只需替换 <code>CLIP Text Encode</code> 中的文本即可重塑画面主题。
- **换风格**：调整 LoRA 的权重或更换新的 LoRA Loader，同时确保与基础模型匹配。
- **换尺寸**：修改 <code>Empty Latent Image</code> 的宽高，并同步调整保存路径中备注的分辨率。

完成上述步骤后，你已经拥有一条可复用的文生图流水线。下一章我们将学习如何把社区共享的工作流改造成自己的日常模板。
